PVE 及 TrueNAS CORE 的安装与简单设置
2023-07-26 14:10  

J4125 自用备份，不是什么详尽的教程。

PVE 更换企业源

付费订阅才能使用企业源，否则得更换非订阅源才能正常更新 PVE。

PVE 关闭 KSM

在内存冗余的情况下，关闭 KSM （内存去重）以节省 CPU 开销。

PVE 删除 local-lvm 并把空间合并至 local

快照等功能对我来说暂时不需要。

PVE 开启 PCI 直通

  • 修改 grub
    vi /etc/default/grub
    GRUB_CMDLINE_LINUX_DEFAULT="quiet amd_iommu=on iommu=pt"
    update-grub

  • 添加内核模块
    vi /etc/modules

    update-initramfs -u -k all

  • 重启

PVE 给垃圾 RTL 8125B 网卡更换官方驱动

默认的驱动是r8169，有断流情况。
建议使用此仓库中的驱动，该驱动启用了TX多队列及RSS并禁用ASPM。
确保 PVE 的软件源没问题。

安装内核头

之后把仓库中的软件包安装上去，安装过程略。
可通过ethtool -i enp3s0查看当前驱动，其中enp3s0为网口。
大概率还是r8169，这样的话还需要屏蔽r8169，更新引导。

重启后可再次通过ethtool -i enp3s0确认驱动；
及通过cat /proc/interrupts | grep -P 'enp3s0|CPU0'查看中断数。
此时可在虚拟机的VirtIO网卡中设置Multiqueue了。
在虚拟机中可使用ethtool -l eth0来验证开启网卡多队列。

PVE 创建虚拟机

在特定情况下，按以下配置创建可提高运行效率。

PVE 给虚拟机开启 virtio-gl

virtio-gl，通常称为 VirGL，是一种用于虚拟机内部的虚拟 3D GPU，可以将工作负载卸载到主机 GPU，而无需特定的驱动程序，也不会完全绑定主机 GPU，从而允许在多个客户机和/或主机之间重用。

需要给 PVE 宿主机安装额外的软件包。
apt install libgl1 libegl1
然后在虚拟机的硬件 -> 显示选择VirGL GPU。

PVE 虚拟机 允许外部 VNC 访问

允许外部的 VNC Viewer 远程连接 VM ID 为 100 的虚拟机，IP 为 PVE 物理机所在 IP，端口为5901。
qm set 101 -args "-vnc :1"

PVE RDM 裸磁盘映射

查看需要映射的磁盘的 ID。
ls -la /dev/disk/by-id/|grep -v dm|grep -v lvm|grep -v part
输出例如：

修改此命令：
qm set <vmid> --scsiX /dev/disk/by-id/xxxxxxx
例如：
qm set <vmid> --scsi2 /dev/disk/by-id/nvme-eui.5cd2e4bf63af0100

TrueNAS CORE 开启 RTL 8125B 网卡支持

最新版已经内置了 RTL 2.5G 网卡的驱动，但存在些许问题，没有默认开启。
修改以下文件；添加：
vi /boot/loader.conf

TrueNAS CORE 简单设置

系统 -> 高级 -> 内核：启用自动调节

TrueNAS CORE 全闪阵列下 设置内存仅缓存元数据

zfs set primarycache=metadata pool/datab

TrueNAS CORE 监狱 pkg 更换中科大源

国外人士可跳过。
创建目录及文件；粘贴以下内容：

pkg update -f

TrueNAS CORE 监狱 开启 SSH

修改以下文件；找到对应项目更改为：
vi /etc/ssh/sshd_config

设置 root 用户密码；SSH 开机自启；启用 SSH。
passwd root

TrueNAS CORE 监狱 编译安装 qBittorrent-Enhanced-Edition

pkg install git pkgconf qt5-core qt5-network qt5-svg qt5-linguisttools qt5-buildtools qt5-qmake boost-all libtorrent-rasterbar qt5-sql openssl
git clone https://github.com/c0re100/qBittorrent-Enhanced-Edition

如果是通过 SCP 协议接收的文件，还需更改权限。
chmod -R +x qBittorrent-Enhanced-Edition

编辑源码添加一行，否则编译报错。
vi qBittorrent-Enhanced-Editionsrc/src.pro
DEFINES += _GNU_SOURCE

编译及安装。

设置开机启动，并将相关配置文件保存到你喜欢的位置。
vi /etc/rc.local
/usr/bin/qbittorrent-nox --webui-port=8080 -d --profile=/root/qbconfig

Linux 虚拟机允许 ssh 连接 root 用户

vi /etc/ssh/sshd_config
PermitRootLogin yes

Linux 虚拟机设置串口输出显示

vi /etc/default/grub
修改GRUB_CMDLINE_LINUX_DEFAULT="quiet中的quiet
GRUB_CMDLINE_LINUX_DEFAULT="console=tty0 console=ttyS0,115200n8"
update-grub

如果提示update-grub不存在，可以手动创建
vi /usr/sbin/update-grub

Linux 虚拟机改回传统网络接口名称

ln -s /dev/null /etc/udev/rules.d/80-net-setup-link.rules
